# M365 Lighthouse Baseline Deployment — Management Templates and Tracking

## Overview

M365 Lighthouse provides **Management Templates** (pre-built security and compliance baselines)
that MSPs can deploy to customer tenants. Each template contains **steps** that map to specific
configurations (CA policies, Defender settings, Intune compliance policies, etc.).

**Base URL**: `https://graph.microsoft.com/beta/tenantRelationships/managedTenants`

---

## List Available Management Templates

```
GET {base}/managementTemplates
?$select=id,displayName,description,category,lastActionDateTime
&$orderby=category asc
```

### Response Structure

```json
{
  "value": [
    {
      "id": "e2cadc41-a08f-45e7-8eb1-942d224dfb9a",
      "displayName": "MFA for Admins",
      "description": "Require multi-factor authentication for all administrators",
      "category": "identity",
      "workloadActions": [
        {
          "actionId": "ca-admin-mfa",
          "category": "conditionalAccess",
          "displayName": "Require MFA for Admins Conditional Access Policy",
          "description": "Creates a CA policy requiring MFA for all admin roles"
        }
      ]
    }
  ]
}
```

### Template Categories

| Category | Description |
|----------|-------------|
| `identity` | Authentication and access (MFA, CA, legacy auth) |
| `device` | Device compliance, Defender, Intune enrollment |
| `data` | Data protection, retention, sensitivity labels |
| `apps` | Application security, app protection policies |
| `infrastructure` | Windows Update, security baselines |

## Get Template Steps

```
GET {base}/managementTemplateSteps?$filter=managementTemplateId eq '{template-id}'
&$select=id,displayName,description,category,templateStepVersion,acceptedVersion
```

## Check Deployment Status Across All Tenants

```
GET {base}/managementTemplateStepTenantSummaries
?$filter=managementTemplateStepId eq '{step-id}'
&$select=tenantId,tenantDisplayName,assignedToTenantCount,deployedToTenantCount,notDeployedToTenantCount,dismissedToTenantCount
```

### Status Aggregation

```json
{
  "value": [
    {
      "tenantId": "customer-tenant-id",
      "tenantDisplayName": "Contoso Customer",
      "managementTemplateStepId": "{step-id}",
      "assignedToTenantCount": 1,
      "deployedToTenantCount": 1,
      "notDeployedToTenantCount": 0
    }
  ]
}
```

## Deployment Status Per Template Per Tenant

```
GET {base}/managementActionTenantDeploymentStatuses
?$filter=tenantId eq '{tenant-id}'
&$select=tenantId,tenantDisplayName,statuses
&$expand=statuses
```

### Status Values Explained

| Status | Meaning | Action |
|--------|---------|--------|
| `compliant` | Deployed and compliant | None needed |
| `notCompliant` | Not yet deployed | Queue for deployment |
| `error` | Deployment failed | Investigate error, retry |
| `dismissed` | MSP dismissed the requirement | Review if intentional |
| `notLicensed` | Tenant lacks license | Upgrade license tier |
| `excluded` | Tenant excluded from template | Confirm exclusion is valid |

## Track Management Actions

Management Actions are individual remediation tasks generated by Lighthouse. They differ from
Management Templates in that they are reactive (based on detected issues) vs. proactive (templates).

### List All Management Actions

```
GET {base}/managementActions
?$select=id,displayName,description,category,workloadActions
```

### List Action Deployment Status

```
GET {base}/managementActionTenantDeploymentStatuses
?$filter=tenantId eq '{tenant-id}'
&$orderby=lastRefreshedDateTime desc
```

## Baseline Deployment Workflow — Step by Step

### Step 1: Identify Applicable Templates

```
GET {base}/managementTemplates
```

Filter by category and check which templates are relevant for the customer's license tier:
- **Business Premium**: MFA baselines, Defender for Business, Intune basic compliance
- **E3**: Everything above + advanced compliance, information protection
- **E5**: All + Defender for Identity, MCAS policies, Advanced Threat Protection

### Step 2: Check Which Tenants Are Non-Compliant

```
GET {base}/managementTemplateStepTenantSummaries
?$filter=managementTemplateStepId eq '{step-id}' and notDeployedToTenantCount gt 0
```

### Step 3: Generate Remediation Tasks

For each non-compliant tenant + template step, generate a remediation item:

1. **Template**: What needs to be deployed
2. **Tenant**: Which customer
3. **Current state**: Why it's not compliant
4. **Required action**: Specific steps to deploy
5. **GDAP role needed**: Which partner role is required
6. **Effort**: Quick / Moderate / Complex

### Step 4: Apply Management Action via Lighthouse

For supported management actions, Lighthouse can apply them directly:

```
POST {base}/managementActions/{actionId}/apply
Content-Type: application/json

{
  "tenantId": "{customer-tenant-id}",
  "tenantGroupId": "{{group-id}}",
  "managementTemplateId": "{template-id}"
}
```

### Step 5: Verify Compliance

Re-query step summaries after deployment. Allow up to 24 hours for Lighthouse data refresh.

## Tracking Template Versions

Templates update over time. Track version alignment:

```
GET {base}/managementTemplateStepVersions
?$filter=managementTemplateStepId eq '{step-id}'
&$select=id,version,displayName,hasHigherPriorityStep,deployedStepCount
&$orderby=version desc
```

When a new version is available, non-upgraded tenants will show as `notCompliant` again.

## Lighthouse Workbooks — Cross-Tenant Analytics

Lighthouse exposes aggregate analytics for cross-tenant reporting:

```
GET {base}/credentialUserRegistrationsSummaries
?$orderby=tenantDisplayName asc
&$select=tenantId,tenantDisplayName,totalUserCount,mfaRegisteredUserCount,
         adminsMfaRegisteredCount,adminsCount,securityDefaultsEnabled,
         mfaConditionalAccessPolicyState
```

Use this data to generate monthly MSP reporting tables showing baseline adoption across all
managed tenants, with trend analysis vs. previous months.
