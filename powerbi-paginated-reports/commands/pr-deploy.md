# Deploy Paginated Report

Generate and optionally execute a deployment script for uploading an .rdl file to a Fabric workspace.

## Instructions

1. Parse arguments:
   - `<path-to-rdl>` — Path to the .rdl file to deploy
   - `--workspace` — Fabric workspace ID (GUID)
   - `--name` — Display name in workspace (defaults to filename without extension)
   - `--overwrite` — Replace existing report with same name (default: abort on conflict)
   - `--bind-gateway` — Gateway ID to bind data sources after upload

2. Verify the .rdl file exists and is valid XML.

3. Read REST API reference:
   - `${CLAUDE_PLUGIN_ROOT}/skills/paginated-reports/references/rest-api.md`
   - `${CLAUDE_PLUGIN_ROOT}/skills/paginated-reports/examples/api-automation.md`

4. Generate a TypeScript deployment script that:
   - Authenticates via MSAL (service principal or device code)
   - Uploads the .rdl via POST /imports
   - Polls import status until complete
   - Optionally binds to gateway
   - Outputs the report URL on success

5. Write the script to `deploy-report.ts` (or user-specified path).

6. If Node.js and required packages are available, offer to execute the script.

## Guidelines

- Never hardcode credentials in the generated script — use environment variables
- Always validate the .rdl is well-formed XML before attempting upload
- Default nameConflict to 'Abort' for safety — only use 'CreateOrOverwrite' with `--overwrite`
- Include error handling with actionable error messages
- Include a dry-run mode that shows what would happen without executing
- Reference the REST API rate limits (50 exports/hour, 5 concurrent)
- Add poll interval with exponential backoff (start 3s, max 30s)
