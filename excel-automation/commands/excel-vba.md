# /excel-vba — Generate VBA Macro (Deprecated)

> **DEPRECATION NOTICE**: VBA macros are a legacy technology. Office Scripts (TypeScript) are the modern replacement supported by Microsoft 365. Use `/excel-script` instead unless you specifically need VBA for desktop-only Excel (Windows/Mac) without Microsoft 365.

## Purpose

Generates VBA macro code for Excel automation. This command exists solely as a fallback for environments where Office Scripts are not available (Excel desktop without Microsoft 365, or `.xlsm` macro-enabled workbooks in legacy workflows).

**This command will only generate VBA when `--legacy-vba` is explicitly included in the argument.** Without it, the command redirects the user to `/excel-script`.

## Instructions

When this command is invoked:

### Without `--legacy-vba` Flag

If the user runs `/excel-vba <description>` without the `--legacy-vba` flag:

1. **Do NOT generate VBA code.**
2. **Display the deprecation notice** and explain why Office Scripts are preferred:
   - Office Scripts run in the cloud (Excel on the web and desktop)
   - TypeScript is safer and more maintainable than VBA
   - Office Scripts integrate with Power Automate for automation
   - No macro security warnings or `.xlsm` requirements
   - Better IDE support and type safety
3. **Suggest the equivalent `/excel-script` command** with the same description.
4. **Ask if the user specifically needs VBA** (e.g., for desktop-only Excel without M365).

### With `--legacy-vba` Flag

If the user runs `/excel-vba <description> --legacy-vba`:

1. **Acknowledge the legacy requirement** and proceed with VBA generation.

2. **Load the office-scripts skill** for context on equivalent modern patterns (useful for generating the Office Script alternative suggestion).

3. **Generate the VBA macro** following these standards:
   - Use explicit `Option Explicit` at the top of every module
   - Declare all variables with explicit types (`Dim x As Long`, not `Dim x`)
   - Use meaningful variable names (not `i`, `j`, `x`)
   - Include error handling with `On Error GoTo ErrorHandler` pattern
   - Use `With` blocks for object property setting to reduce repetition
   - Avoid `Select` and `Activate` — work with objects directly
   - Use `Application.ScreenUpdating = False` for performance
   - Use `Application.Calculation = xlCalculationManual` for bulk operations
   - Restore application state in the error handler / cleanup section
   - Add comments explaining non-obvious logic

4. **Structure the VBA code** as:
   ```vba
   Option Explicit

   Sub MainProcedure()
       ' Purpose: [description]
       ' Author: Generated by Claude
       ' Date: [date]

       On Error GoTo ErrorHandler

       ' Performance optimization
       Application.ScreenUpdating = False
       Application.Calculation = xlCalculationManual

       ' --- Main logic here ---

   Cleanup:
       Application.ScreenUpdating = True
       Application.Calculation = xlCalculationAutomatic
       Exit Sub

   ErrorHandler:
       MsgBox "Error " & Err.Number & ": " & Err.Description, vbCritical, "Error"
       Resume Cleanup
   End Sub
   ```

5. **Also provide the Office Script equivalent** (or a note explaining what the equivalent `/excel-script` command would be) so the user can migrate when ready.

6. **Write the VBA code** to a `.bas` file (standard VBA export format) or display it inline with instructions for importing into the VBA editor.

7. **Include migration guidance**:
   - Which VBA patterns map to which Office Script APIs
   - What changes when moving from VBA to TypeScript
   - Common gotchas in migration (no `ActiveSheet`, no `Select`, different range syntax)

## VBA to Office Script Quick Reference

| VBA Pattern | Office Script Equivalent |
|------------|-------------------------|
| `ActiveSheet` | `workbook.getActiveWorksheet()` |
| `Sheets("Name")` | `workbook.getWorksheet("Name")` |
| `Range("A1:B10")` | `sheet.getRange("A1:B10")` |
| `Range("A1").Value` | `sheet.getRange("A1").getValue()` |
| `Range("A1:B10").Value` | `sheet.getRange("A1:B10").getValues()` |
| `Cells(row, col)` | `sheet.getCell(row, col)` |
| `Selection` | *(not available — use explicit ranges)* |
| `MsgBox` | `console.log()` |
| `Application.ScreenUpdating` | *(not needed — batched by default)* |
| `On Error GoTo` | `try { } catch (e) { }` |
| `For Each cell In Range` | `for (let row of values) { }` |
| `ListObjects("Table1")` | `sheet.getTable("Table1")` |
| `Dim x As Long` | `let x: number` |
| `Dim s As String` | `let s: string` |
| `Sub Name()` | `function main(workbook: ExcelScript.Workbook)` |

## Checklist Before Output

- [ ] `--legacy-vba` flag is present (otherwise redirect to `/excel-script`)
- [ ] `Option Explicit` is at the top
- [ ] All variables have explicit type declarations
- [ ] Error handling with `On Error GoTo ErrorHandler` is included
- [ ] `ScreenUpdating` and `Calculation` are managed for performance
- [ ] Application state is restored in the cleanup section
- [ ] No `Select` or `Activate` usage (work with objects directly)

## Example Usage

```bash
# This will NOT generate VBA — redirects to /excel-script
/excel-vba Format the sales report with headers and charts

# This WILL generate VBA (with deprecation notice)
/excel-vba Format the sales report with headers and charts --legacy-vba

# Legacy automation requirement
/excel-vba --legacy-vba Create a macro that imports CSV files from a folder into separate sheets

# Desktop-only Excel scenario
/excel-vba --legacy-vba Build a userform for data entry with validation and database insert
```
